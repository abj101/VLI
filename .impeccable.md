# Impeccable — JARVIS (VLI)

Design context for UI work in this repository. Update this file when product direction changes.

## Design Context

### Users

The app serves people who are **comfortable with their computers** and want a **clean, elegant, and enjoyable** way to run automations and drive basic system actions. It is **unapologetically nerdy**: power-user affordances are welcome as long as the surface stays legible. The core job is to **run predetermined, branching action sequences quickly**, triggered by **voice commands** (with hotkey and editor support as part of the same workflow).

### Brand Personality

**Classy, purposeful, focused.** The interface should feel considered and calm, not busy or apologetic. Every visual choice should earn its place; whimsy appears where it supports the task, not as decoration for its own sake.

### Aesthetic Direction

- **North star:** **Luxury quiet** with **selective expressiveness** — restrained default chrome, clear hierarchy, and memorable moments where state changes matter (for example listening, match, execution feedback on the HUD).
- **Surfaces — frosted glass default:** Overlays and the **HUD** use **frosted glass** with **subtle transparency** (not heavy wash), restrained blur, and a soft edge highlight. Prioritize readability first: translucency only where text remains comfortably legible.
- **Reference language:** **Apple macOS** — continuity of shape, **purposeful glass** (material depth, blur, and layering where it aids comprehension), disciplined **typography**, and **careful color** (neutrals with intentional accent use). This is inspiration for *craft*, not pixel-for-pixel imitation of Apple apps.
- **Anti-patterns:** Avoid a **Windows 11** decorative / “settings app” flavor; avoid **Cursor-like** saturated dev-tool palettes; avoid **generic shadcn** template layouts (same card grids, same purple accents, same component soup).
- **Theme (current phase):** Ship **light and dark** as equal citizens. Dark mode should feel **premium night glass**: slightly off-black, deep, and calm, never flat mid-gray.
- **Accessibility:** Meet **WCAG 2.1 AA** for contrast and interactive targets where applicable.
- **Motion:** Keep motion **sparse and functional**; no requirement for a dedicated in-app reduced-motion toggle beyond honoring system preferences if motion is added later.

### Design Principles

1. **Voice-first clarity** — Surfaces (HUD, editor, settings) make **state and outcome** obvious at a glance; transcription and command feedback stay the hero where relevant.
2. **Quiet luxury, loud moments** — Default UI stays **calm and spacious**; **expressive** treatment reserved for **meaningful transitions** (listen / match / execute / done), not global animation noise.
3. **Glass with intent** — HUD glass stays **subtle**: enough translucency to feel native, enough tint/contrast to keep text readable on varied wallpapers.
4. **Apple-adjacent craft, not cliché** — Favor **rounded continuity**, **typographic discipline**, and **tinted neutrals** over flat gray slabs; still **distinctly JARVIS**, not a stock “Big Sur clone.”
5. **Anti-generic product UI** — Reject **Win11** ornament, **IDE neon**, and **template-dashboard** patterns; custom composition and spacing over repeated card-icon-title grids.
6. **AA non-negotiable** — Light and dark themes both ship with **contrast-safe** text, borders, and controls.

#### Transparency, blur, readability targets

- **HUD glass treatment (subtle):**
  - `--hud-bg-alpha: 0.78`
  - `--hud-backdrop-blur: 18px`
  - `--hud-saturation: 1.08` (if supported)
  - `--hud-border: 1px solid color-mix(in oklch, var(--color-divider), white 45%)`
- **Settings panes (mostly opaque, reference-like):**
  - `--settings-pane-alpha: 0.94`
  - `--settings-group-alpha: 0.98`
  - `--settings-backdrop-blur: 0px` (no decorative blur in standard settings sections)
- **Dark HUD glass (night glass tuning):**
  - `--hud-dark-bg-alpha: 0.64`
  - `--hud-dark-backdrop-blur: 24px`
  - `--hud-dark-saturation: 1.12` (if supported)
  - `--hud-dark-border: 1px solid color-mix(in oklch, var(--color-dark-divider), white 18%)`
  - `--hud-dark-inner-glow: inset 0 1px 0 color-mix(in oklch, var(--color-dark-text-primary), transparent 88%)`
- **Dark settings panes (cohesive with light structure):**
  - `--settings-dark-pane-alpha: 0.9`
  - `--settings-dark-group-alpha: 0.94`
  - `--settings-dark-backdrop-blur: 6px` (only where depth separation is needed)
- **Minimum contrast requirements:**
  - Body text on any surface: **>= 4.5:1**
  - Large text (>= 18px regular or >= 14px bold): **>= 3:1**
  - Interactive/focus indicator against adjacent color: **>= 3:1**
  - For dark mode body text target **>= 7:1** where feasible to keep parity with light-mode readability.

#### Cross-theme cohesion rules

- Keep **identical spacing, type scale, and component geometry** across themes; only color/material shifts.
- Keep semantic color mapping stable: background → surface → elevated → divider → text.
- Maintain the same accent hue family in both themes; only adjust lightness/chroma for contrast.
- Avoid "double-dark" layering where both parent and child surfaces collapse into the same value.
- Validate legibility over wallpaper/noisy content any time blur/transparency is used.

#### Corner radius guidance

- Apple does not publish one global macOS window corner-radius constant in public HIG/API docs.
- Community measurement discussions commonly cite **~16pt** for modern rounded window shells; treat this as a practical reference token, not an Apple-guaranteed constant.
- Use this radius scale for phase 1:
  - `--radius-window-outer: 16px`
  - `--radius-panel-large: 14px`
  - `--radius-panel-medium: 12px`
  - `--radius-control: 10px`
  - `--radius-chip-pill: 999px`
- **Relationship rule:** nested surfaces step down by 2px per layer to preserve concentric feel.

#### What this supersedes

- Supersedes prior "heavy diffusion / high-opacity milky glass" direction for HUD.
- New default is subtle transparency with strict readability constraints.
- Supersedes prior "light theme only" direction; dark mode is now required and quality-equal.

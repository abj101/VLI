# Impeccable — JARVIS (VLI)

Design context for UI work in this repository. Update this file when product direction changes.

## Design Context

### Users

The app serves people who are **comfortable with their computers** and want a **clean, elegant, and enjoyable** way to run automations and drive basic system actions. It is **unapologetically nerdy**: power-user affordances are welcome as long as the surface stays legible. The core job is to **run predetermined, branching action sequences quickly**, triggered by **voice commands** (with hotkey and editor support as part of the same workflow).

### Brand Personality

**Classy, purposeful, focused.** The interface should feel considered and calm, not busy or apologetic. Every visual choice should earn its place; whimsy appears where it supports the task, not as decoration for its own sake.

### Aesthetic Direction

- **North star:** **Luxury quiet** with **selective expressiveness** — restrained default chrome, clear hierarchy, and memorable moments where state changes matter (for example listening, match, execution feedback on the HUD).
- **Surfaces — frosted glass default:** Overlays and the **HUD** use **frosted glass** — **heavy backdrop blur**, a **high-opacity tinted fill** (milky / diffuse, not a thin wash), and a **subtle edge highlight** — so depth reads intentional while staying readable. Tune blur, tint strength, and border **per theme** for **WCAG AA**; avoid decorative blur with no content behind it.
- **Reference language:** **Apple macOS** — continuity of shape, **purposeful glass** (material depth, blur, and layering where it aids comprehension), disciplined **typography**, and **careful color** (neutrals with intentional accent use). This is inspiration for *craft*, not pixel-for-pixel imitation of Apple apps.
- **Anti-patterns:** Avoid a **Windows 11** decorative / “settings app” flavor; avoid **Cursor-like** saturated dev-tool palettes; avoid **generic shadcn** template layouts (same card grids, same purple accents, same component soup).
- **Theme:** **Follow the system** preference with **first-class light and dark** modes. Visual bias toward **glass and depth** that reads as refined in **both** themes — not “dark only = premium.” Palette stays **primarily neutral** in light and dark; no mandatory fixed brand colors beyond what the themes require.
- **Accessibility:** Meet **WCAG 2.1 AA** for contrast and interactive targets where applicable.
- **Motion:** Keep motion **sparse and functional**; no requirement for a dedicated in-app reduced-motion toggle beyond honoring system preferences if motion is added later.

### Design Principles

1. **Voice-first clarity** — Surfaces (HUD, editor, settings) make **state and outcome** obvious at a glance; transcription and command feedback stay the hero where relevant.
2. **Quiet luxury, loud moments** — Default UI stays **calm and spacious**; **expressive** treatment reserved for **meaningful transitions** (listen / match / execute / done), not global animation noise.
3. **Glass with intent** — **Frosted glass** on the HUD and similar overlays means **strong diffusion** (blur + opaque-leaning tint), not faint transparency. It **separates the panel from the wallpaper** without sacrificing legibility; tune in **both** themes; respect **`prefers-contrast: more`** with an opaque fallback when transparency would hurt readability.
4. **Apple-adjacent craft, not cliché** — Favor **rounded continuity**, **typographic discipline**, and **tinted neutrals** over flat gray slabs; still **distinctly JARVIS**, not a stock “Big Sur clone.”
5. **Anti-generic product UI** — Reject **Win11** ornament, **IDE neon**, and **template-dashboard** patterns; custom composition and spacing over repeated card-icon-title grids.
6. **AA non-negotiable** — Light and dark themes both ship with **contrast-safe** text, borders, and controls; test critical paths in each mode.

### Engineering notes (from existing product docs)

- **HUD + editor** split: overlay for capture/feedback; **separate editor window** for command CRUD. Editor: **no emoji**; **geometric** icon language; **10px** corner radius and **44px** row rhythm where applicable per `BrainStorm.md` unless a deliberate design pass updates those tokens coherently.
- **Stack:** Tauri 2, React, Framer Motion, Zustand — motion should remain **light** per direction above.
